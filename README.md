# Rate Limiter(fixed window)

**Паттерн Rate Limiter** используется для управления нагрузкой на систему, ограничивая 
количество запросов, которые клиент может отправить на сервер за определённый период времени.

**Fixed window** - алгоритм ограничения скорости с фиксированным окном. Он разделяет время на фиксированные 
интервалы (окна) и ограничивает количество запросов в каждом из них.

**Применение**: алгоритм используется в сценариях, где нужно соблюдать ограничение скорости в определённые временные интервалы, 
например, в ограничении скорости API и процессах аутентификации пользователей.

### Принцип работы
- Определение интервала окна — например, 1 секунда, 1 минута или 1 час.
- Отслеживание запросов — используется счётчик, который фиксирует количество запросов в текущем окне.
- Сброс счётчика — в конце окна счётчик сбрасывается до нуля и начинается счёт снова для нового окна.
- Проверка ограничения скорости — счётчик сравнивается с допустимым лимитом. Если счётчик превышает лимит, дополнительные запросы отбрасываются до начала следующего окна.
